## Github

While I share projects here on GitHub, most of my projects happen in GitLab.

I’ve gravitated toward GitLab for its early and more integrated approach to the full development lifecycle, including:

- Built-in CI/CD pipelines with strong native support  
- First-class support for private repositories and internal projects  
- Integrated tooling for code review, automation, and deployment  
- A more unified platform for managing code, infrastructure, and workflows
- (Much of which github NOW supports... but it didn't always)

That said, GitHub plays an important role in my workflow.

Projects here are typically:

- Easier to share due to GitHub’s broader adoption and visibility  
- Intended for collaboration with the wider community  
- Required to be hosted on GitHub for compatibility with certain tools, platforms, or integrations  

In practice, I use both platforms intentionally depending on the context.
